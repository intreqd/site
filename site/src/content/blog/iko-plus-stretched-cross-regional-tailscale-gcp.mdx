In this installment of the **IKO Plus** series, we are going to get a little bit "stretched." We'll look at how to deploy an InterSystems IrisCluster that spans multiple Google Cloud regions (e.g., US and UK) using **Tailscale** to provide a unified, secure overlay network between our Kubernetes clusters.

### The Problem: Cross-Regional High Availability

Traditionally, stretching a database cluster across regions involves complex VPC peering, VPN gateways, and careful CIDR management. We want the performance and simplicity of the InterSystems Kubernetes Operator (IKO) but with the resilience of a multi-region footprint.

### The Solution: Tailscale Overlay

Tailscale allows us to create a secure, flat mesh network between nodes in different regions without worrying about underlying firewalls or complex routing. By installing Tailscale on our Kubernetes nodes, we can allow the IKO to manage mirror pairs across the Atlantic.

![Tailscale Overlay Network](https://community.intersystems.com/sites/default/files/inline/images/images/image(11785).png)

### The Setup

We'll start with two Kubernetes clusters in Google Cloud Platform (GCP):
- **Region A**: `us-east1`
- **Region B**: `europe-west2` (UK)

Each cluster will have Tailscale installed, and we'll use Tailscale's subnet routing to expose the pod and service CIDRs to each other.

### IKO Topology

Using the IKO, we define a single `IrisCluster` resource. The magic happens in the `topology` section where we specify `preferredZones` to ensure our mirror pair is distributed across our regional footprints.

```yaml
apiVersion: intersystems.com/v3
kind: IrisCluster
metadata:
  name: stretched-iris
spec:
  topology:
    dataMaster:
      preferredZones: ["us-east1-a"]
    dataSlave:
      preferredZones: ["europe-west2-a"]
```

### Deploying the Stretched Cluster

Once the networking is established, we apply our configuration.
