In this installment of **IKO Plus**, we're talking about **Secrets Management**. Shhhhhhhhh! 🤫 Dealing with passwords, certificates, and API keys in Kubernetes can be a security nightmare if not handled correctly. We want to avoid hardcoding secrets in our manifests or keeping them in plain-text Kubernetes secrets.

### The Solution: External Secrets Operator (ESO)

The **External Secrets Operator** integrates Kubernetes with external secret management systems like Google Secret Manager, AWS Secrets Manager, or HashiCorp Vault. It reads information from a external API and automatically injects the values into a Kubernetes Secret.

### Step 1: Defining the Secret Store

First, we define a `SecretStore` that tells ESO how to connect to our cloud secret manager (in this case, Google Secret Manager).

```yaml
# secretstore.yaml snippet
apiVersion: external-secrets.io/v1beta1
kind: SecretStore
metadata:
  name: gcp-sm-store
spec:
  provider:
    gcpsm:
      projectID: my-gcp-project
```

### Step 2: Creating an ExternalSecret

Next, we create an `ExternalSecret` resource. This tells ESO which specific secret to fetch from the cloud and what the target Kubernetes secret should be named.

![ESO in Headlamp](https://community.intersystems.com/sites/default/files/inline/images/images/image(12156).png)

```yaml
# externalsecret.yaml
apiVersion: external-secrets.io/v1beta1
kind: ExternalSecret
metadata:
  name: external-containers-pull-secret
spec:
  secretStoreRef:
    name: gcp-sm-store
    kind: SecretStore
  target:
    name: containers-pull-secret
    template:
      type: kubernetes.io/dockerconfigjson
      data:
        .dockerconfigjson: "{{ .dockerconfig | toString }}"
  data:
    - secretKey: dockerconfig
      remoteRef:
        key: ikoplus-eso-gcp
```

### Step 3: Integrating with IKO

Now we simply update our IKO Helm chart `values.yaml` to use the secret that ESO is managing for us.

```yaml
# values.yaml
imagePullSecrets:
  - name: containers-pull-secret
```

### Summary

The **External Secrets Operator** is a powerful tool for any production IRIS workload on Kubernetes. It allows you to leverage the full security and auditing capabilities of cloud secret managers while keeping your Kubernetes deployments clean and declarative.
